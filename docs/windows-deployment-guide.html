1. Prerequisites

Before starting, ensure you have the following installed on your Windows system:

Node.js (v20 or higher)

1 Download Node.js

Visit https://nodejs.org and download the LTS version (v20.x or higher).

2 Install Node.js

Run the installer and follow the wizard. Accept default options and ensure "Add to PATH" is checked.

3 Verify Installation

Open Command Prompt or PowerShell and run:

node --version
npm --version

You should see version numbers (e.g., v20.10.0 and 10.2.4).

Git (Optional but Recommended)

Download from https://git-scm.com/download/win

2. PostgreSQL Installation

1 Download PostgreSQL

Visit https://www.postgresql.org/download/windows/

Click "Download the installer" and select the latest version (PostgreSQL 16+).

2 Run the Installer

Double-click the downloaded .exe file (requires administrator privileges).

3 Installation Settings
  • Installation Directory: Keep default (C:\Program Files\PostgreSQL\16)
  • Components: Select all (PostgreSQL Server, pgAdmin 4, Command Line Tools)
  • Data Directory: Keep default
  • Password: Set a secure password for the postgres superuser
  • Port: Keep default 5432
  • Locale: Keep default
Important: Remember your PostgreSQL superuser password! You'll need it for database setup.
4 Add PostgreSQL to PATH
  1. Press Win + R, type sysdm.cpl, press Enter
  2. Go to Advanced tab ‚Üí Environment Variables
  3. Under System variables, select Path ‚Üí Edit
  4. Click New and add: C:\Program Files\PostgreSQL\16\bin
  5. Click OK on all dialogs
5 Verify PostgreSQL Installation

Open a new Command Prompt (important: must be new window after PATH change) and run:

psql --version

Expected output: psql (PostgreSQL) 16.x

3. Database Setup

1 Open PostgreSQL Command Line

Open Command Prompt as Administrator and connect to PostgreSQL:

psql -U postgres

Enter your superuser password when prompted.

2 Create Database and User

Run these SQL commands in the psql prompt:

PostgreSQL Commands
-- Create the database
CREATE DATABASE licenseiq;

-- Create a dedicated user (replace 'YourSecurePassword123!' with your own)
CREATE USER licenseiq_user WITH PASSWORD 'YourSecurePassword123!';

-- Grant all privileges
GRANT ALL PRIVILEGES ON DATABASE licenseiq TO licenseiq_user;

-- Connect to the new database
\c licenseiq

-- Grant schema privileges
GRANT ALL ON SCHEMA public TO licenseiq_user;
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO licenseiq_user;
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO licenseiq_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO licenseiq_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON SEQUENCES TO licenseiq_user;

-- Enable required extensions
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
CREATE EXTENSION IF NOT EXISTS "vector";

-- Exit psql
\q
Note: The vector extension is required for AI embeddings. If it fails, you may need to install pgvector separately.

Installing pgvector Extension (if needed)

If the CREATE EXTENSION vector command fails:

  1. Download pgvector from GitHub releases
  2. Follow the Windows installation instructions in the repository
  3. Or use the Stack Builder (installed with PostgreSQL) to add extensions

5. Environment Configuration

Copy from Replit: Your API keys are already configured in your Replit project. To copy them:
  1. Open your Replit project
  2. Click the Tools menu in the left sidebar
  3. Click Secrets
  4. Copy each secret value to your local .env file
1 Create Environment File

Create a file named .env in the project root directory with the following content:

.env
# ============================================
# DATABASE CONFIGURATION (Local PostgreSQL)
# ============================================
# Format: postgresql://USERNAME:PASSWORD@HOST:PORT/DATABASE
DATABASE_URL=postgresql://licenseiq_user:YourSecurePassword123!@localhost:5432/licenseiq

# Individual PostgreSQL Settings
PGHOST=localhost
PGPORT=5432
PGUSER=licenseiq_user
PGPASSWORD=YourSecurePassword123!
PGDATABASE=licenseiq

# ============================================
# APPLICATION SETTINGS
# ============================================
NODE_ENV=development
PORT=5000
SESSION_SECRET=generate-a-random-32-character-string-here

# ============================================
# AI SERVICES - COPY FROM REPLIT SECRETS
# ============================================
# Groq API Key (REQUIRED for AI contract analysis)
# Get from: https://console.groq.com/keys
# Copy from Replit Secrets ‚Üí GROQ_API_KEY
GROQ_API_KEY=gsk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# OpenAI API Key (Optional - for enhanced features)
# Get from: https://platform.openai.com/api-keys
# Copy from Replit Secrets ‚Üí OPENAI_API_KEY
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# HuggingFace API Key (REQUIRED for embeddings/RAG)
# Get from: https://huggingface.co/settings/tokens
# Copy from Replit Secrets ‚Üí HUGGINGFACE_API_KEY
HUGGINGFACE_API_KEY=hf_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# ============================================
# EMAIL CONFIGURATION (Optional)
# ============================================
# Zoho Mail credentials for sending notifications
ZOHO_EMAIL=
ZOHO_PASSWORD=
Security: Never commit the .env file to version control. Add it to .gitignore.
2 Update Database URL

Replace YourSecurePassword123! with the actual password you set for licenseiq_user in Step 3.

3 Copy API Keys from Replit

In your Replit project, go to Tools ‚Üí Secrets and copy these values:

Replit Secret Name .env Variable Required?
GROQ_API_KEY GROQ_API_KEY Yes - AI Analysis
HUGGINGFACE_API_KEY HUGGINGFACE_API_KEY Yes - Embeddings
OPENAI_API_KEY OPENAI_API_KEY Optional
4 Generate Session Secret

Generate a random 32+ character string for SESSION_SECRET. You can use PowerShell:

[System.Convert]::ToBase64String((1..32 | ForEach-Object { Get-Random -Minimum 0 -Maximum 256 }))

7. Database Migration

1 Push Database Schema

Run the Drizzle migration to create all database tables:

npm run db:push
Note: If you encounter any errors, try running with force flag: 
npx drizzle-kit push --force
2 Verify Tables Were Created

Connect to the database and check tables:

psql -U licenseiq_user -d licenseiq -h localhost

-- List all tables
\dt

-- Exit
\q

You should see tables like users, contracts, companies, business_units, locations, etc.

10. Troubleshooting

Common Issues

Error: ECONNREFUSED - Cannot connect to database

Solution:

  1. Ensure PostgreSQL service is running: 
    sc query postgresql-x64-16
  2. Start the service if stopped: 
    net start postgresql-x64-16
  3. Verify your DATABASE_URL in .env is correct
Error: Password authentication failed

Solution:

  1. Verify the password in DATABASE_URL matches what you set for the user
  2. Reset user password in psql: 
    psql -U postgres
ALTER USER licenseiq_user WITH PASSWORD 'NewPassword123!';
Error: Extension "vector" is not available

Solution:

Install pgvector extension:

  1. Download from pgvector GitHub
  2. Follow Windows installation instructions
  3. Restart PostgreSQL service
Error: Port 5000 is already in use

Solution:

  1. Find the process using port 5000: 
    netstat -ano | findstr :5000
  2. Kill the process (replace PID with actual number): 
    taskkill /PID [PID] /F
  3. Or change port in .env: PORT=3000
Error: npm install fails with permission errors

Solution:

  1. Run Command Prompt as Administrator
  2. Or try: npm cache clean --force

Logs Location

Application logs are printed to the console. For PM2 logs:

pm2 logs licenseiq

11. API Keys Setup

LicenseIQ uses free AI services. Here's how to get your API keys:

Groq API Key (Required for AI Analysis)

1 Sign Up for Groq

Visit https://console.groq.com

2 Generate API Key

Go to API Keys section ‚Üí Create new key ‚Üí Copy the key

Free Tier: Groq offers generous free usage with their llama-3.3-70b-versatile model.

HuggingFace API Key (Required for Embeddings)

1 Sign Up for HuggingFace

Visit https://huggingface.co/join

2 Generate Token

Go to Settings ‚Üí Access Tokens ‚Üí New Token ‚Üí Create (read permissions)

OpenAI API Key (Optional - for enhanced features)

1 Sign Up for OpenAI

Visit https://platform.openai.com

2 Generate API Key

Go to API Keys ‚Üí Create new secret key

OpenAI is a paid service. The application works without it using Groq as the primary AI provider.
